Security Analysis Mobile Security Architecture

Developing for mobile platforms like the ArtGalley system requires distinct security strategies compared to web apps. Our architecture addresses the following:

 Input Validation: All inventory fields (Price, Quantity) use strict numeric constraints to prevent SQL injection and data corruption. Data Transmission: The app interacts with the remote ArtGalley REST API using encrypted protocols, ensuring inventory data is not exposed during transit. Privacy Requirements

To comply with ULO4 requirements, this mobile application ensures:

  • Principle of Least Privilege: Only necessary inventory data is requested from the server.
  • Unique Identification: The system enforces unique item names to maintain database integrity and prevent record overwriting.
  • Local Session Security: Temporary data held in memory is cleared when the mobile application session is terminated.
